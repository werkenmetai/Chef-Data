> "Elke ondernemer in Nederland kan in natuurlijke taal
> met zijn boekhouding praten, zonder technische kennis."

---

## Waarom Dit Ertoe Doet

### Het Probleem

1. **Ondernemers begrijpen hun cijfers niet**
   - 70% van MKB'ers checkt maximaal 1x per maand hun financiën
   - Boekhoudtaal is ontoegankelijk
   - Software is complex (40+ knoppen in Exact)

2. **Boekhouders zijn overbelast**
   - Simpele vragen kosten tijd
   - "Hoeveel BTW moet ik betalen?" → email → wachten → antwoord
   - Inefficiënt voor beide partijen

3. **AI is er, maar niet gekoppeld**
   - ChatGPT kan rekenen, maar heeft geen data
   - Boekhouddata zit opgesloten in silo's
   - Geen standaard protocol (tot MCP)

### Onze Oplossing

```
┌─────────────────────────────────────────────────────────────┐
│  "Hoeveel omzet had ik vorige maand?"                       │
│                      ↓                                      │
│  [Praat met je Boekhouding - MCP Server]                   │
│                      ↓                                      │
│  "Je omzet in december was €42.350, 12% meer dan november" │
└─────────────────────────────────────────────────────────────┘
```

## Strategic Principles

### 1. AI-First, Always
```
Vraag bij elke taak: "Kan een agent dit doen?"
Ja → Delegeer aan agent
Nee → Evalueer of het nodig is
```

### 2. Revenue Before Features
```
Geen feature bouwen zonder bewijs dat klanten het willen.
"Would you pay for this?" is de enige vraag die telt.
```

### 3. Git als Command Center
```
Alles traceerbaar in Git.
Beslissingen, planning, code, content.
Transparant en reproduceerbaar.
```

### 4. Minimal Viable Effort
```
Wat is het minimum om de volgende stap te zetten?
Perfectie is de vijand van progressie.
```

### 5. Compounding Assets
```
Bouw dingen die in waarde groeien over tijd:
- SEO content (rankt beter met leeftijd)
- Knowledge base (agents worden slimmer)
- Partnerships (referrals groeien)
- Brand (trust compounds)
```
